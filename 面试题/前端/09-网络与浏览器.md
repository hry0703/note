## 1. HTTP/HTTPS

**问：HTTP 和 HTTPS 的区别？HTTPS 的加密过程？**

**答：**

| 对比 | HTTP | HTTPS |
|------|------|-------|
| 端口 | 80 | 443 |
| 传输 | 明文 | 加密 |
| 身份 | 无 | 证书验证 |
| 完整性 | 无校验 | 防篡改 |

**HTTPS = HTTP + SSL/TLS**，在应用层和传输层之间加了一层安全层。

**握手过程（简化）**：
1. 客户端发送支持的加密套件、随机数
2. 服务端返回证书、公钥、随机数
3. 客户端验证证书，用公钥加密预主密钥发送
4. 服务端用私钥解密，得到预主密钥
5. 双方用三个随机数生成会话密钥，后续通信用对称加密

## 2. 跨域与解决方案

**问：什么是跨域？有哪些解决方式？**

**答：**

**跨域**：协议、域名、端口任一不同，即跨域。浏览器同源策略限制跨域请求。

**解决方案**：

| 方案 | 说明 |
|------|------|
| **CORS** | 服务端设置 `Access-Control-Allow-Origin` 等响应头 |
| **代理** | 开发时用 devServer proxy，请求同源，由服务端转发 |
| **JSONP** | 用 `<script>` 跨域，仅支持 GET，已较少使用 |
| **postMessage** | 跨窗口通信 |
| **Nginx 反向代理** | 同代理思路 |

```javascript
// 服务端 CORS
res.setHeader('Access-Control-Allow-Origin', '*');
res.setHeader('Access-Control-Allow-Methods', 'GET, POST, OPTIONS');
res.setHeader('Access-Control-Allow-Headers', 'Content-Type');
```

**预检请求**：非简单请求（如带自定义头、PUT）会先发 OPTIONS 预检，通过后再发实际请求。

## 3. 浏览器缓存（强缓存、协商缓存）

**问：强缓存和协商缓存的区别？相关头有哪些？**

**答：**

### 强缓存

不发请求，直接用本地缓存。由 `Expires` 或 `Cache-Control` 控制。

```
Cache-Control: max-age=3600      // 3600 秒内用缓存
Cache-Control: no-cache          // 可用缓存，但需向服务器验证
Cache-Control: no-store          // 不缓存
Expires: Thu, 01 Jan 2026 00:00:00 GMT  // 绝对时间，优先级低于 Cache-Control
```

### 协商缓存

向服务器确认是否可用缓存。由 `Last-Modified`/`If-Modified-Since` 或 `ETag`/`If-None-Match` 控制。

```
Last-Modified: 文件修改时间
If-Modified-Since: 客户端上次收到的 Last-Modified

ETag: 文件唯一标识（更精确）
If-None-Match: 客户端上次收到的 ETag
```

**流程**：先看强缓存 → 过期则发请求 → 带 If-Modified-Since/If-None-Match → 304 则用本地缓存，200 则返回新内容。

## 4. 从输入 URL 到页面展示的流程

**问：简述从输入 URL 到页面展示的完整流程？**

**答：**

1. **URL 解析**：解析协议、域名、路径等
2. **DNS 查询**：域名 → IP（浏览器缓存 → 系统缓存 → 路由器 → ISP → 递归查询）
3. **建立 TCP 连接**：三次握手
4. **HTTPS**：TLS 握手
5. **发送 HTTP 请求**：请求行、头、体
6. **服务器处理**：返回响应
7. **浏览器解析**：
   - HTML → DOM 树
   - CSS → CSSOM 树
   - DOM + CSSOM → 渲染树
   - 布局（Layout）
   - 绘制（Paint）
   - 合成（Composite）
8. **加载子资源**：JS、图片等，可能触发重排、重绘
9. **连接结束**：四次挥手（或 Keep-Alive 复用）
