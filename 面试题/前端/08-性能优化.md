# 性能优化 面试题

---

## 1. Core Web Vitals（LCP、CLS、INP）

**问：Core Web Vitals 有哪些指标？如何优化？**

**答：**

Google 提出的核心 Web 体验指标：

| 指标 | 全称 | 含义 | 优化方向 |
|------|------|------|----------|
| **LCP** | Largest Contentful Paint | 最大内容绘制时间 | 首屏关键资源加载、服务端渲染、CDN |
| **FID/INP** | First Input Delay / Interaction to Next Paint | 首次输入延迟 | 减少长任务、拆分 JS |
| **CLS** | Cumulative Layout Shift | 累积布局偏移 | 图片/广告预留尺寸、字体加载策略 |

**LCP 优化**：预加载关键资源、优化图片、减少阻塞渲染的 JS/CSS。

**CLS 优化**：给图片/视频设 `width/height`，字体用 `font-display: optional` 或预加载。

## 2. 首屏加载优化

**问：如何优化首屏加载速度？**

**答：**

1. **减少资源体积**：代码压缩、Tree Shaking、图片压缩、Gzip/Brotli
2. **减少请求数**：合并文件、雪碧图、内联关键 CSS
3. **加快解析**：关键 CSS 内联、非关键 CSS 异步加载
4. **预加载**：`<link rel="preload">` 关键资源
5. **代码分割**：路由懒加载，首屏只加载必要代码
6. **SSR/SSG**：服务端渲染或静态生成，减少白屏
7. **CDN**：静态资源走 CDN，就近访问
8. **HTTP/2**：多路复用，减少连接开销

## 3. 代码分割与懒加载

**问：如何做代码分割和懒加载？**

**答：**

**代码分割**：按路由或按功能拆成多个 chunk，按需加载。

```javascript
// 路由懒加载（React）
const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));

<Suspense fallback={<Loading />}>
  <Routes>
    <Route path="/" element={<Home />} />
    <Route path="/about" element={<About />} />
  </Routes>
</Suspense>
```

```javascript
// 动态 import（Vue）
const routes = [
  {
    path: '/home',
    component: () => import('./views/Home.vue'),
  },
];
```

**图片懒加载**：`loading="lazy"` 或 Intersection Observer 在进入视口时再加载。

## 4. 资源预加载（preload、prefetch）

**问：preload 和 prefetch 的区别？**

**答：**

| 类型 | 用途 | 优先级 |
|------|------|--------|
| **preload** | 当前页面马上要用的资源 | 高，立即加载 |
| **prefetch** | 下一页可能用到的资源 | 低，空闲时加载 |

```html
<!-- 预加载关键字体 -->
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

<!-- 预取下一页资源 -->
<link rel="prefetch" href="next-page.js">
```

**dns-prefetch / preconnect**：提前做 DNS 解析或建立连接，减少后续请求延迟。

## 5. 图片优化

**问：有哪些图片优化手段？**

**答：**

1. **格式**：WebP、AVIF 替代 JPEG/PNG，体积更小
2. **压缩**：TinyPNG、imagemin 等工具压缩
3. **尺寸**：按展示尺寸输出，避免大图小用
4. **懒加载**：`loading="lazy"` 或 Intersection Observer
5. **响应式**：`srcset`、`sizes` 按设备加载合适尺寸
6. **雪碧图**：小图标合并，减少请求
7. **CDN**：图片走 CDN，可配合裁剪、压缩等处理

```html
<img
  src="image.webp"
  srcset="image-400.webp 400w, image-800.webp 800w"
  sizes="(max-width: 600px) 400px, 800px"
  loading="lazy"
  alt="描述"
>
```

## 6. 减少 HTTP 请求

**问：如何减少 HTTP 请求？**

**答：**

1. **合并**：合并 JS/CSS 文件、雪碧图
2. **内联**：关键 CSS 内联，减少首屏请求
3. **缓存**：强缓存、协商缓存，减少重复请求
4. **HTTP/2**：多路复用，同一连接多请求
5. **资源精简**：移除无用代码、按需加载
6. **本地存储**：localStorage 缓存接口数据（注意过期策略）
