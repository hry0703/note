# V8 与内存 面试题

---

## 1. 垃圾回收机制

**问：V8 的垃圾回收机制是怎样的？**

**答：**

V8 采用**分代式垃圾回收**，将内存分为新生代和老生代。

### 新生代（存活时间短的对象）

- **算法**：Scavenge（复制算法）
- **空间**：分为 From 和 To 两个等大区域
- **过程**：From 中存活对象复制到 To，清空 From，交换 From 和 To
- **晋升**：经历过多次回收仍存活，或 To 空间超过 25%，晋升到老生代

### 老生代（存活时间长的对象）

- **算法**：Mark-Sweep（标记清除）+ Mark-Compact（标记整理）
- **过程**：
  1. 标记：遍历可达对象做标记
  2. 清除：回收未标记对象
  3. 整理：移动存活对象，减少碎片（可选）

### 增量标记

为避免全堆标记造成长时间停顿，V8 使用**增量标记**，将标记工作拆成多步，与 JS 执行交替进行。

## 2. 内存泄漏排查与避免

**问：如何排查和避免内存泄漏？**

**答：**

### 常见内存泄漏场景

1. **全局变量**：未用 `var/let/const` 声明的变量挂在 window 上
2. **未清理的定时器**：`setInterval` 未 `clearInterval`
3. **闭包引用**：闭包持有大对象，且未及时释放
4. **DOM 引用**：移除 DOM 后仍保留其引用
5. **事件监听未移除**：组件销毁时未 `removeEventListener`

### 排查方法

1. **Chrome DevTools → Memory**：拍堆快照，对比前后差异
2. **Performance Monitor**：观察 JS heap size 是否持续增长
3. **Allocation instrumentation**：查看对象分配和保留路径

### 避免建议

```javascript
// 定时器记得清理
const id = setInterval(fn, 1000);
componentWillUnmount() {
  clearInterval(id);
}

// 事件监听记得移除
element.addEventListener('click', handler);
element.removeEventListener('click', handler);

// 闭包注意及时解除引用
let bigData = getBigData();
function process() { /* 使用 bigData */ }
process();
bigData = null; // 用完后解除引用
```

## 3. 堆与栈

**问：堆和栈的区别？JavaScript 中如何体现？**

**答：**

| 对比项 | 栈（Stack） | 堆（Heap） |
|--------|-------------|------------|
| 存储内容 | 基本类型、引用地址、执行上下文 | 引用类型（对象、数组等） |
| 分配方式 | 自动分配、固定大小 | 动态分配、较大空间 |
| 访问速度 | 快 | 相对慢 |
| 生命周期 | 随函数执行结束释放 | 由 GC 回收 |

```javascript
let a = 1;           // a 和 1 在栈中
let b = { x: 1 };    // b 和引用地址在栈，{ x: 1 } 在堆
let c = [1, 2, 3];   // c 和引用在栈，数组在堆
```

**执行上下文栈**：函数调用时，会创建执行上下文并压入调用栈，执行完后弹出。
