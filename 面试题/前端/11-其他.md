## 1. Git 常用命令与工作流

**问：Git 常用命令有哪些？你们的工作流是怎样的？**

**答：**

### 常用命令

```bash
# 基础
git clone <url>
git add .
git commit -m "msg"
git push origin <branch>
git pull origin <branch>

# 分支
git branch <name>
git checkout -b <name>
git merge <branch>
git rebase <branch>

# 撤销
git reset --soft HEAD~1   # 撤销 commit，保留修改
git reset --hard HEAD~1   # 撤销 commit，丢弃修改
git checkout -- <file>    # 撤销工作区修改
git stash                 # 暂存
git stash pop             # 恢复

# 查看
git status
git log --oneline
git diff
```

### 常见工作流

- **Git Flow**：master + develop + feature 分支，适合长期维护
- **GitHub Flow**：master 为主，feature 分支开发，PR 合并
- **Trunk Based**：主干开发，短期分支，适合 CI/CD

## 2. 项目难点复盘（STAR 法则）

**问：介绍一下你做过最有挑战的项目？**

**答：**

用 **STAR 法则** 组织回答：

- **S（Situation）**：项目背景、业务目标
- **T（Task）**：你在其中的职责
- **A（Action）**：具体做法、技术选型、遇到问题如何解决
- **R（Result）**：可量化的结果（性能提升、体验改善、稳定性等）

**示例**：  
「在 XX 项目中，首屏加载较慢（S），我负责前端性能优化（T）。通过路由懒加载、图片懒加载、接口合并、CDN 等手段（A），首屏时间从 3.2s 降到 1.5s，LCP 提升约 50%（R）。」
